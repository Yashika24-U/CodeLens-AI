What is Singleton Pattern?

👉 Simple definition:

Singleton ensures that only ONE instance of a class/object exists in the entire application, and provides a global access point to it.

🔥 Real-life example
Database connection
Logger
Config manager

👉 You don’t want multiple copies — just one shared instance
1. Singleton pattern
No matter how many times you create → same object

🧠 Key Characteristics
Only one instance
Global access
Shared state
🔥 Have YOU used Singleton?

👉 YES — but indirectly

In your Sequelize setup:

const sequelize = new Sequelize(config);
db.sequelize = sequelize;
🧠 Why this behaves like Singleton?

Because of Node.js module caching

require('./models')

👉 Runs only once
👉 Returns same object everywhere

So:
One sequelize instance
Shared across app

👉 That’s Singleton behavior

⚠️ Important Interview Truth

👉 You did NOT write a strict Singleton class

But you can say:

“It behaves like a Singleton due to module caching.”

🏭 1. Factory Pattern (simple + powerful)
🧠 Definition

A Factory is a function (or class) that creates and returns objects, instead of you creating them directly.

💻 In YOUR code
require(file)(sequelize, DataTypes)

Inside each model file:

module.exports = (sequelize, DataTypes) => {
  const User = sequelize.define("User", {});
  return User;
};
🧠 Why this is a Factory?

👉 Because:

You call a function
It creates a model
It returns that model

✔️ You are NOT doing:

const User = new User(); ❌

✔️ Instead:

const User = createUserModel(sequelize); ✅
🎯 Real-world meaning

“Give me inputs → I’ll create the object for you”

🔥 Why Factory is useful
Centralizes object creation
Makes code flexible
Hides creation logic
🎯 Interview line

“Each Sequelize model file acts as a factory function that creates and returns a model instance when provided with sequelize and DataTypes.”

📦 2. Registry Pattern (very important)
🧠 Definition

A Registry is a central place where you store and access shared objects.

💻 In YOUR code
db[model.name] = model;
🧠 What is db?
db = {
  User: UserModel,
  Post: PostModel
}

👉 This is your registry

🧠 Why this matters

Instead of:

const User = require('./user');
const Post = require('./post');

👉 You just do:

db.User
db.Post
🎯 Real-world meaning

“Store everything in one place so anyone can access it easily”

🔥 Why Registry is useful
Central access point
Avoids repeated imports
Cleaner architecture
🎯 Interview line

“All models are stored in a centralized registry object (db), allowing easy access and management across the application.”

🔥 Factory vs Registry (VERY IMPORTANT)
Pattern	Role
Factory	Creates objects
Registry	Stores objects
🧠 In YOUR flow:
Factory → creates model
Registry → stores model
💥 Visual Flow
model.js (Factory)
        ↓
creates User model
        ↓
index.js
        ↓
db["User"] = model  (Registry)
🎯 FINAL INTERVIEW ANSWER (STRONG)

“In my Sequelize setup, each model file acts as a factory function that creates a model instance when invoked. These models are then stored in a centralized registry object (db), which allows easy access and management across the application.”
